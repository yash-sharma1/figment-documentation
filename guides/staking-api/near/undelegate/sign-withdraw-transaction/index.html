Sign the Withdrawal Transaction

Sign the withdrawal transaction using the transaction_payload you received in response to submitting the withdraw data.

Locate the transaction_payload in the response from the previous step. It will look something like this:

"transaction_payload": "12000000736c6174652d64656d6f2e746573746e657400415d57cdf21d28e35e0cfa864e895c18b4bc6a711642d38eb6762af81ec1cf3907cc1038fc5700001600000030316e6f64652e706f6f6c2e663836333937332e6d3084edd29289b3a590be582317bb0061016a8df058b7c29a0278f5b82f86c283010100000002080000007769746864726177270000007b22616d6f756e74223a223235303030303030303030303030303030303030303030303030227d00e057eb481b000000000000000000000000000000000000"
important

Always confirm the details of the transaction before signing the payload.

Figment has a tool for signing and decoding Staking API payloads, see the guide Signing Transactions with Figment's npm Package to continue.

Transactions can also be signed using the NearJS API or Fireblocks.

Sign the transaction and move on to the next step. A signed transaction payload for NEAR will look something like this:
