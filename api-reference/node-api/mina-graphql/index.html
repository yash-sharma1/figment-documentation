Node API - Mina GraphQL

Authenticationâ€‹

API calls require authentication with API key via Authorization request header or URL property. Calls are made to the Mina Graphql Node API endpoint, hosted at https://mina-mainnet--graphql.datahub.figment.io/graphql

Here's some boilerplate to get you started.
  • Fetch with Header
  • Fetch with Parameter
  • CURL with Header
  • CURL with Parameter
fetch("https://mina-mainnet--graphql.datahub.figment.io/graphql", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Authorization" : "<api_key>",
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
fetch("https://mina-mainnet--graphql.datahub.figment.io/apikey/<api_key>/graphql", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
curl -X POST 'https://mina-mainnet--graphql.datahub.figment.io/graphql' \
--header 'Content-Type: application/json' \
--header 'Authorization: <api_key>' \
--data '{
  // JSON Payload
}'
curl -X POST 'https://mina-mainnet--graphql.datahub.figment.io/apikey/<api_key>/graphql' \
--header 'Content-Type: application/json' \
--data '{
  // JSON Payload
}'

network status

Check out the node/network status.

SpecificationView Specification

Parameters

  • None

Response

  • Return the node/network status data including syncStatus, stateHash, numAccounts, chainId, commitId, catchupStatus and blockchainLength.
Request Body (try it)
query status {
  daemonStatus {
    syncStatus
    stateHash
    numAccounts
    chainId
    commitId
    catchupStatus
    blockchainLength
  }
}

current block

Pull the latest canonical block and its information.

SpecificationView Specification

Parameters

  • None

Response

  • Return latest canonical block data including bestChain, creator, stateHash, stateHashField, protocolState, transactions and coinbase.
Request Body (try it)
query block {
  bestChain(maxLength: 1) {
    creator
    stateHash
    stateHashField
    protocolState {
      blockchainState {
        date
        snarkedLedgerHash
      }
      previousStateHash
      consensusState {
        blockHeight
        blockchainLength
        epoch
        slot
        totalCurrency
      }
    }
    transactions {
      userCommands {
        amount
        fee
        from
        hash
      }
      coinbase
    }
  }
}

account details

Pull the account details for the given account(publicKey: " ").

SpecificationView Specification

Parameters

GraphQL query params -

  • account(publicKey: " ") - Public key of the account for which you want to fetch the details.

Response

  • Return the account details data including delegate, balance, blockHeight, liquid, locked, stateHash, total, unknown, delegators, publicKey, stakingActive, votingFor and isDisabled.
Request Body (try it)
query accDetails {
  account(publicKey: "B62qix9vooX5NqJYo8nT6xWqCeQu5AJoS1ng6FRnUpVAra6PAZZ1CU4") {
    delegate
    balance {
      blockHeight
      liquid
      locked
      stateHash
      total
      unknown
    }
    delegators {
      publicKey
    }
    publicKey
    stakingActive
    votingFor
    isDisabled
  }
}
