Transaction Search API - NEAR Protocol

Authentication​

API calls require authentication with API key via Authorization request header or URL property. Calls are made to the Near Protocol Transaction Search API endpoint, hosted at https://near--indexer.datahub.figment.io

Here's some boilerplate to get you started.
  • Fetch with Header
  • Fetch with Parameter
  • CURL with Header
  • CURL with Parameter
fetch("https://near--indexer.datahub.figment.io", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Authorization" : "<api_key>",
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
fetch("https://near--indexer.datahub.figment.io", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
curl -X POST 'https://near--indexer.datahub.figment.io' \
--header 'Content-Type: application/json' \
--header 'Authorization: <api_key>' \
--data '{
  // JSON Payload
}'
curl -X POST 'https://near--indexer.datahub.figment.io' \
--header 'Content-Type: application/json' \
--data '{
  // JSON Payload
}'

Parameters

Query string parameters -

  • block_hash - string Filter by block hash.
  • block_height - string Filter by block height.
  • sender - Account which sent the transaction.
  • receiver - Account which was the transaction recipient.
  • start_date - string Search range start time.
  • end_date - string Search range end time.
  • start_height - string Search range start time.
  • end_height - string Search range end time.
  • limit - Limit results. Max is 100.

Response

Returns transaction details.
