Staking API - Avalanche

Authenticationâ€‹

API calls require authentication with API key via Authorization request header or URL property. Calls are made to the Avalanche Staking API endpoint, hosted at https://avalanche-slate.datahub.figment.io

Here's some boilerplate to get you started.
  • Fetch with Header
  • Fetch with Parameter
  • CURL with Header
  • CURL with Parameter
fetch("https://avalanche-slate.datahub.figment.io/api/v1/flows", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Authorization" : "<api_key>",
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
fetch("https://avalanche-slate.datahub.figment.io/apikey/{{apikey}}/api/v1/flows", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
curl -X POST 'https://avalanche-slate.datahub.figment.io/api/v1/flows' \
--header 'Content-Type: application/json' \
--header 'Authorization: <api_key>' \
--data '{
  // JSON Payload
}'
curl -X POST 'https://avalanche-slate.datahub.figment.io/apikey/{{apikey}}/api/v1/flows' \
--header 'Content-Type: application/json' \
--data '{
  // JSON Payload
}'

Available Methods

Create New Delegation Flow

To initiate the staking process, create a new flow with a POST request to /api/v1/flows.

Request Body 
{
  "flow": {

    "chain_code": "testnet",

SpecificationView Specification

Parameters

  • flow â€” object
    • network_code â€” string Network this flow operates on (ex. avalanche) (required).
    • chain_code â€” string Chain this flow operates on (ex. testnet) (required).
    • operation â€” string The operation to perform (ex. staking) (required).
    • version â€” string API version to use (ex. v1) (optional).

Response

  • id â€” string ID of the flow.
  • state â€” string The current state of the flow.
  • actions â€” array It includes the name & inputs of all next possible actions.
  • create_delegate_tx â€” Use this action to generate a delegation transaction to stake AVAX.
    • delegator_address â€” The delegating address, where AVAX is being staked from. This must be a P-chain address.
    • validator_address â€” Figment's validator address.
    • amount â€” The amount of AVAX to be delegated.
    • start_time â€” Optional timestamp for when the delegation will be active.
    • end_time â€” Optional timestamp for when the delegation will end.
    • memo â€” A plaintext memo to identify the transaction.
  • data â€” object Delegation flow & transaction data.
