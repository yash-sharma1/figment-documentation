Authenticationâ€‹

API calls require authentication with API key via Authorization request header or URL property. Calls are made to the Solana Staking API endpoint, hosted at https://solana-slate.datahub.figment.io/api/v1/flows

Here's some boilerplate to get you started.
  • Fetch with Header
  • Fetch with Parameter
  • CURL with Header
  • CURL with Parameter
fetch("https://solana-slate.datahub.figment.io/api/v1/flows", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Authorization" : "<api_key>",
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
fetch("https://solana-slate.datahub.figment.io/apikey/<api_key>/api/v1/flows", {
    method: 'POST', // can also be 'GET', 'PUT' or any appropriate method
    headers: {
        "Content-Type": "application/json" // if method is 'POST'
    },
    body: JSON.stringify({
      // JSON Payload
    })
})
curl -X POST 'https://solana-slate.datahub.figment.io/api/v1/flows' \
--header 'Content-Type: application/json' \
--header 'Authorization: <api_key>' \
--data '{
  // JSON Payload
}'

--header 'Content-Type: application/json' \

Available Methods

New Staking Flow

Create a new Staking flow with a POST request to /api/v1/flows.

SpecificationView Specification

Parameters

  • flow - object
    • network_code - string Network this flow operates on (i.e., near) (required)
    • chain_code - string Chain this flow operates on (i.e., mainnet, testnet) (required)
    • operation string The operation to -perform (i.e., staking) (required)
    • version - string API version to use (ex. v1) (optional)

Response

  • id - string Id of the newly created flow.
  • state - string The current state of the flow i.e., initialized.
  • actions - array The actions array contains the name of the next action in the flow and details of the expected inputs in the data field.

Assign Stake Account

Use this action if you already have an inactive stake account you want to activate.

SpecificationView Specification

After collecting the required inputs, send a PUT request to /api/v1/flows/[:flow_id]/next to move on to the next step in the flow.

Parameters

  • name - string Name of the action to execute (required).
  • inputs - array
    • stake_account_pubkey - string The address of the stake account to be activated (required).
    • funding_account_pubkey - string The main wallet SOL will be originating from (required).

Response

  • id - string ID of the newly created flow.
  • state - string Current state of the flow (i.e., delegate_tx_signature).
  • actions - array The actions array contains the name of the next action in the flow and details of the expected inputs in the data field.
Request Body 
{
  "name": "assign_stake_account",
  "inputs": {
    "stake_account_pubkey": "",
    "funding_account_pubkey": ""
  }
}
Example Response
""

Create New Stake Account

Use this action if you do not have an inactive stake account and want to create a new one.

SpecificationView Specification

After collecting the required inputs, PUT /api/v1/flows/[:flow_id]/next to move on to the next step in the flow.

Parameters

  • name - string Name of the action to execute (required).
  • inputs - array.
    • funding_account_pubkey - string The main wallet SOL will be originating from (required).
    • stake_authority_pubkey - string Grants authority to sign certain staking relation transactions to the provided address. Defaults to funding_account_pubkey (optional).
    • withdraw_authority_pubkey - string Grants authority to sign certain withdrawal-related transactions to the provided address. Defaults to funding_account_pubkey (optional).
  • amount - number The amount of SOL to be sent to the Stake Account. This field should be in full SOL, not in lamports (i.e., if you want to stake 5 SOL, return "amount": 5.0). Staking API will translate the amount to lamports so you do not have to worry about the conversion factor (required).

Response

Note that in this response you can find the unsigned, serialized transaction in data[create_stake_account_transaction][raw].

  • id - string Id of the newly created flow.
  • state - string Current state of the flow i:e; delegate_tx_signature
  • actions - array The actions array contains the name of the next action in the flow and details of the expected inputs in the data field.

Sign Transfer Transaction

Submit a signed transaction payload ready to be broadcast to the network.

SpecificationView Specification

Parameters

  • name - string Name of the action you want to execute (i.e., sign_transfer_tx).
  • inputs - array
    • transaction_payload - string Transaction payload can be found in the response after creating the transaction. (required). OR
    • signatures - array An array of signatures. Staking API will sign the payload if this is provided (optional).

Response

With the signed transaction provided, Staking API will broadcast it to the network. As the transaction progresses towards confirmation, the returned state will be:

  • transfer_tx_broadcasting - The transaction has been broadcast and is waiting for confirmation.
  • confirmed - The transfer transaction has been confirmed.
